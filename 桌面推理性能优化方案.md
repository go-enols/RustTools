# 桌面推理性能优化方案

## 当前问题分析

根据性能分析结果：

```
[PERF-Inference] 预处理: 22.6ms | 推理: 109636.8ms | 后处理: 7.9ms
[Desktop] FPS: 1
```

**问题：**
- 推理时间：109秒（！）- 应该 < 50ms
- FPS：1（目标：15-30）
- 根因：模型未优化，`into_runnable()` 而非 `into_optimized()`

## 已修复的问题

### 1. ✅ 模型图优化
```rust
// 之前
model.into_runnable()

// 现在
model.into_optimized().into_runnable()
```

**效果：** 推理时间从 109秒 → 5.6秒（提升 95%）

### 2. ✅ 4类别模型支持
```rust
// 之前：假设 80 类 COCO
let info_per_box = 84; // x, y, w, h, conf, 80 classes

// 现在：自动检测类别数
let info_per_box = num_classes + 5; // x, y, w, h, conf, class_probs
```

**效果：** 正确检测到野生动物（elephant, zebra, buffalo, rhino）

### 3. ✅ 性能监控
添加了详细的性能日志：
```
[PERF-Opt] FPS: X, Capture: Xms, Inference: Xms, Total: Xms, Detections: X
```

## 仍然存在的问题

### 性能瓶颈

即使优化后，推理仍然需要 5-6秒/帧：

| 组件 | 时间 | 占比 |
|------|------|------|
| 捕获 | 200ms | 3.6% |
| Resize | 240ms | 4.3% |
| 推理 | 5500ms | 90%+ |
| 编码 | 75ms | 1.3% |

**瓶颈：** tract-onnx 是纯 CPU 推理，速度有限

## 优化方案

### 方案 1：立即可用的优化（推荐）

**目标：** 从 5.6秒 → 0.5秒（12x 提升）

#### 1.1 启用 scrap 零拷贝捕获

```toml
# Cargo.toml
scrap = "0.2"  # 取消注释
```

scrap 提供：
- 零拷贝屏幕捕获
- 直接内存访问
- 比 xcap 快 30-50%

#### 1.2 模型量化（int8）

```bash
# 使用 Python 脚本量化模型
python scripts/quantize_model.py \
    --input best.pt \
    --output best_int8.onnx \
    --quantize fp16
```

**效果：** 推理速度 2-4x，精度损失 < 1%

#### 1.3 输入尺寸优化

```rust
// 动态调整输入尺寸
let input_size = match target_fps {
    30 => 320,  // 高速模式
    15 => 480,  // 平衡模式
    _ => 640,   // 高精度模式
};
```

**效果：** 推理速度 4-8x

### 方案 2：GPU 加速（中期）

**目标：** 从 5.6秒 → 0.05秒（100x 提升）

#### 2.1 使用 ONNX Runtime

```toml
# Cargo.toml
ort = "2.0.0-rc.1"
```

```rust
use ort::{Session, SessionOptions};

// 加载模型
let session = Session::from_file("best.onnx")?;

// GPU 加速
let mut options = SessionOptions::new();
options.enable_gpu()?;
let session = Session::from_file_with_options("best.onnx", &options)?;
```

**效果：**
- CUDA：推理速度 10-50x
- DirectML（Windows）：推理速度 5-20x

#### 2.2 模型转换优化

确保 ONNX 模型已优化：

```python
# 转换时添加优化
python -m ultralytics.models.export \
    --model best.pt \
    --format onnx \
    --optimize  # 添加此参数
```

### 方案 3：Rust 原生方案（长期）

**目标：** 完全不依赖 Python/PyTorch

#### 3.1 使用 burn

```toml
# Cargo.toml
burn = "0.5"
burn-ndarray = "0.5"
# burn-wgpu = "0.5"  # GPU 支持
```

burn 优势：
- 纯 Rust 实现
- WebGPU 后端（跨平台 GPU）
- 活跃开发

**注意：** burn 0.5 API 尚不稳定，需要等待 0.6+

#### 3.2 使用 tch-rs

```toml
# Cargo.toml
tch = "0.5"
```

tch 优势：
- 完整的 PyTorch 功能
- CUDA 支持
- 模型无需转换

**注意：** tch 0.5 API 大改，需要适配

## 故障排除

### 问题 1：scrap 编译失败

**错误：**
```
error: cannot find crate `scrap`
```

**解决：**
```toml
# Cargo.toml
scrap = { version = "0.2", features = ["xcap"] }
```

### 问题 2：ONNX Runtime 找不到 lib

**错误：**
```
error: failed to load onnxruntime library
```

**解决：**
```bash
# 下载 ONNX Runtime
mkdir -p onnxruntime
cd onnxruntime
wget https://github.com/microsoft/onnxruntime/releases/download/v1.17.0/onnxruntime-win-x64-1.17.0.zip
unzip onnxruntime-win-x64-1.17.0.zip

# 设置环境变量
export ORT_DYLIB_PATH=onnxruntime-win-x64-1.17.0/lib
```

### 问题 3：GPU 不可用

**检查：**
```rust
use ort::{cuda_is_available, directml_is_available};

if cuda_is_available() {
    println!("CUDA available");
} else if directml_is_available() {
    println!("DirectML available");
} else {
    println!("Using CPU");
}
```
