### 主要瓶颈

**推理时间 5.6秒的原因：**
1. **tract-onnx 是纯 CPU 实现** - 无法利用 GPU
2. **模型较大** - 36MB (YOLOv8m/l，而非 YOLOv8n)
3. **缺乏硬件加速** - 没有 CUDA/DirectML 支持

## 推荐实施步骤

### 立即可用的优化（今天）

**目标：** 从 5.6秒 → 0.5秒（11x 提升）

1. **启用 scrap 零拷贝捕获**
   ```toml
   # Cargo.toml 中启用
   scrap = "0.2"
   ```
   
   scrap 提供：
   - 零拷贝屏幕捕获（比 xcap 快 30-50%）
   - 直接内存访问

2. **使用更小的模型**
   ```bash
   # 下载 YOLOv8n（3-6MB）
   python -c "from ultralytics import YOLO; m = YOLO('yolov8n.pt'); m.export(format='onnx')"
   ```
   
   效果：推理速度 3-5x

3. **调整输入尺寸**
   ```rust
   // 动态调整
   let input_size = match target_fps {
       30 => 320,  // 高速模式
       15 => 480,  // 平衡模式
       _ => 640,   // 高精度模式
   };
   ```
   
   效果：推理速度 4-8x（但精度降低）

### 中期优化（1-3天）

**目标：** 从 0.5秒 → 0.05秒（10x 提升）

**添加 ONNX Runtime 支持：**

```toml
# Cargo.toml
ort = "2.0.0-rc.1"
```

```rust
// 使用 ONNX Runtime 进行 GPU 推理
use ort::{Session, SessionOptions};

let mut options = SessionOptions::new();
options.enable_gpu()?;  // 自动选择 CUDA/DirectML
let session = Session::from_file_with_options("best.onnx", &options)?;
```

效果：
- CUDA：10-50x 提升
- DirectML（Windows）：5-20x 提升

### 长期优化（1-2周）

**目标：** 完全不依赖外部工具

1. **迁移到 burn（当 0.6+ 稳定后）**
   - 等待 burn API 稳定
   - 使用 WebGPU 后端
   - 完全 Rust 实现

2. **模型量化**
   ```bash
   python -m ultralytics.models.export \
       --model best.pt \
       --format onnx \
       --half  # FP16 量化
   ```
   
   效果：2-4x 提升
