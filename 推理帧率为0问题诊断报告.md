# 推理帧率为0问题诊断报告

## 问题诊断结果

经过Python脚本分析,你的YOLO模型输出格式如下:

```
模型文件: C:\Users\25751\Desktop\african-wildlife\yolo11n.onnx
模型输出: [1, 84, 8400]
格式: YOLOv8
```

### 输出格式说明

- **84** = 4(bbox坐标) + 80(COCO类别)
- **8400** = 检测框数量
- 这是**YOLOv8标准格式**

### 当前代码问题

从日志中观察到的问题:
```
[Desktop] Detected 4 objects
  - Class 8102 at (45, 38, 141, 87) conf 616.69
  - Class 6328 at (45, 38, 141, 87) conf 636.57
  - Class 795 at (45, 38, 141, 87) conf 637.28
  - Class 8065 at (45, 38, 141, 87) conf 638.88
```

**问题分析:**
1. ❌ Class ID过大 (应该是 0-3,实际是 795-8102)
2. ❌ Confidence过大 (应该是 0-1,实际是 616-638)
3. ❌ FPS为1而不是15

### 根本原因

模型输出的是**原始logits**(未经激活的原始分数),需要:

1. **类别分数需要sigmoid激活** - 将原始logits转换为0-1之间的概率
2. **Class ID异常** - 可能是模型权重问题或数据格式问题

## 解决方案

### 方案1: 重新导出4类野生动物模型 (推荐)

如果你的野生动物模型确实只有4个类别(elephant, buffalo, rhino, zebra),需要重新导出:

```python
from ultralytics import YOLO

# 加载你的4类野生动物模型
model = YOLO('你的野生动物模型权重.pt')

# 导出为ONNX格式
model.export(format='onnx', imgsz=640, verbose=False)
```

### 方案2: 修复现有代码 (快速测试)

如果暂时不想重新导出模型,可以修改代码应用sigmoid:

```rust
// 在 desktop_capture.rs 的 run_inference 函数中
// 在第301行附近,读取类别分数后添加sigmoid:

for c in 0..num_classes {
    let score = output_data[[0, c + 4, i]];
    let sigmoid_score = 1.0 / (1.0 + (-score).exp()); // sigmoid
    if sigmoid_score > max_score {
        max_score = sigmoid_score;
        max_class = c;
    }
}
```
