## 功能描述

实现 YOLO 模型训练功能，支持：
- 启动/停止/暂停/恢复训练
- 实时显示训练进度和指标
- 支持 CPU/GPU 训练
- 训练结果保存和模型管理

## 技术方案

### 方案选择

| 方案 | 训练 | 部署 | 优点 | 缺点 |
|------|------|------|------|------|
| CLI + Rust推理 | 需要 pip ultralytics | 纯 Rust | 实现简单，功能完整 | 需要 Python 环境 |
| burn 训练 | 纯 Rust | 纯 Rust | 部署简单 | 没有成熟实现 |
| tch-rs | 纯 Rust | 纯 Rust | 可用 PyTorch 训练 | 需要 PyTorch |

**当前采用**: 方案 A (CLI + Rust推理)

**原因**:
1. ultralytics 是最成熟完整的 YOLO 训练方案
2. Rust YOLO 训练库目前没有成熟实现
3. 部署只需 `pip install ultralytics`

### CLI 命令格式

```bash
yolo detect train data={project_path}/data.yaml model={base_model} \
    epochs={epochs} batch={batch_size} imgsz={image_size} \
    device={device_id} workers={workers}
```

### 训练数据格式

**项目目录结构** (与 ultralytics 兼容):
```
project_path/
├── project.yaml          # 自定义项目配置
├── data.yaml             # ultralytics 训练配置 (运行时生成)
├── images/
│   ├── train/           # 训练图片
│   └── val/             # 验证图片
├── labels/
│   ├── train/           # YOLO 格式标注
│   └── val/
├── models/              # 保存的模型
├── runs/                # ultralytics 训练输出
└── weights/             # 模型权重
```

**标注格式** (YOLO 标准格式):
```
class_id x_center y_center width height
0 0.123456 0.234567 0.089012 0.123456
```

### data.yaml 生成

训练开始前自动生成:
```yaml
path: /path/to/project
train: images/train
val: images/val
nc: 2
names: ['class1', 'class2']
```
