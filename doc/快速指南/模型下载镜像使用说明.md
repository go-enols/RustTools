# 模型下载镜像优化 - 快速指南

## 解决的问题

### 问题描述
- 前端直接使用 GitHub 链接下载模型
- 国内用户访问 GitHub 速度极慢（经常几 KB/s）
- 导致模型下载失败或等待时间过长

## 解决方案

### 镜像源优先级
系统自动选择最快的镜像源下载：

1. **ghproxy 镜像** ⭐ 推荐
   - 国内访问 GitHub 的代理
   - 速度：通常 5-10 MB/s
   - 稳定性：极佳

2. **HuggingFace 国内镜像**
   - HuggingFace 官方镜像站
   - 速度：通常 3-5 MB/s
   - 稳定性：良好

3. **GitHub 直连** （备用）
   - 原始地址
   - 速度：通常 0-1 MB/s（国内）
   - 稳定性：差

## 使用方法

### 自动下载
1. 选择要训练的模型（YOLO11、YOLOv10 等）
2. 点击"开始训练"按钮
3. 如果模型不存在，系统自动从镜像源下载
4. 下载完成后自动开始训练

### 手动下载
1. 如果自动下载失败，点击"手动下载"按钮
2. 系统会重新尝试从镜像源下载
3. 如果仍然失败，会显示错误信息

### 查看下载进度
- 弹窗显示实时下载进度
- 显示下载速度和百分比
- 下载完成后自动关闭

## 模型存储位置

### Windows
```
C:\Users\<用户名>\.cache\ultralytics\
```

### Linux/Mac
```
~/.cache/ultralytics/
```

### 查看已下载的模型
```bash
# Windows
dir %USERPROFILE%\.cache\ultralytics\

# Linux/Mac
ls -lh ~/.cache/ultralytics/
```

## 常见问题

### Q: 为什么下载速度还是很慢？
A: 可能是网络问题或镜像源暂时不可用。系统会自动尝试下一个镜像源。

### Q: 下载失败怎么办？
A: 
1. 检查网络连接
2. 稍后重试（镜像源可能暂时不可用）
3. 手动下载模型文件，放入缓存目录

### Q: 如何手动下载模型？
A:
1. 点击"手动下载"按钮
2. 系统会从镜像源重新下载
3. 或手动访问镜像源下载：
   - ghproxy: https://mirror.ghproxy.com/https://github.com/ultralytics/assets/releases/download/v0.0.0/
   - HuggingFace: https://hf-mirror.com/ultralytics/assets/releases/download/v0.0.0/

### Q: 模型文件有多大？
A: 不同模型大小不同：
- YOLO11n: ~5 MB
- YOLO11s: ~10 MB
- YOLO11m: ~30 MB
- YOLO11l: ~50 MB
- YOLO11x: ~100 MB
- YOLOv8x: ~50 MB

### Q: 可以下载后离线使用吗？
A: 可以。模型下载后会保存在缓存目录，后续训练直接使用本地文件，无需重新下载。

## 技术细节

### 模型检测流程
```
1. 前端调用 checkModel()
2. 后端检查 ~/.cache/ultralytics/{model_name}
3. 如果存在且文件大小 > 1MB，直接使用
4. 如果不存在或文件无效，调用 downloadModel()
```

### 镜像下载流程
```
1. 尝试 ghproxy 镜像
2. 如果失败，尝试 HuggingFace 镜像
3. 如果失败，尝试 GitHub 直连
4. 所有镜像失败，返回错误
```

### 文件验证
- 检查文件大小（必须 > 1MB）
- 确保文件完整
