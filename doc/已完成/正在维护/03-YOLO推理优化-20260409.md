## 功能描述

对项目中的桌面推理和视频推理进行优化，结合 Rust YOLO 最佳实践，提升推理性能和实时性。

## 当前系统状态

### ✅ 已实现的优化

1. **纯 Rust 推理引擎**
   - 框架: `tract-onnx` 0.22.0
   - 模型支持: ONNX 格式
   - 后端: CPU 模式

2. **桌面捕获模块**
   - 捕获库: `xcap` 0.1
   - 位置: `src-tauri/src/modules/yolo/services/desktop_capture.rs`
   - 功能: 多显示器支持、FPS 限制、实时检测

3. **视频推理模块**
   - 提取工具: ffmpeg
   - 推理库: `tract-onnx`
   - 位置: `src-tauri/src/modules/yolo/services/video_inference.rs`
   - 功能: 帧提取、批量处理、进度回调

4. **前后端集成**
   - 通信方式: Tauri invoke/events
   - 默认行为: 使用纯 Rust 推理

### ⚠️ 当前限制

1. **CPU-only 模式** - GPU 加速需要额外配置
2. **性能** - CPU 模式下约 10-20 FPS
3. **未使用字段** - DesktopSession 有未使用字段（已标记 #[allow(dead_code)]）

## 技术细节

### 模型格式

- **支持**: ONNX (.onnx)
- **不支持**: PyTorch (.pt)
- **转换**: `yolo export model=xxx.pt format=onnx`

### Tract-ONNX 后端选择

```rust
// CPU 后端（默认）
.into_runnable()

// 优化后的 CPU 后端
.into_optimized()?

// CUDA 后端（需要 feature）
.with_backend(tract_onnx::cuda::backend())?
```

### 内存优化

```rust
// 使用内存池
let buffer = self.memory_pool.lock().await;
buffer.input_buffer.clear();
buffer.input_buffer.resize(3 * 640 * 640, 0.0);
```

### 并行处理

```rust
use rayon::prelude::*;

let results: Vec<_> = frames
    .par_iter()
    .map(|img| engine.detect(img, confidence))
    .collect();
```
