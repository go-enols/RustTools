# Python 环境检测缓存机制 - 使用说明

## 功能简介

实现了 Python 环境检测的缓存机制，用户进入首页时会**优先使用缓存的环境信息**，避免每次都重新检测，提升应用响应速度。

## 核心变化

### ✅ 优化前
```
用户进入首页
  ↓
立即执行 Python 环境检测（耗时 2-5 秒）
  ↓
显示检测结果
```
- **问题**: 每次进入都要等待检测，体验不佳
- **原因**: 没有缓存机制

### ✅ 优化后
```
用户进入首页
  ↓
尝试获取缓存的环境信息
  ↓（有缓存）
直接显示缓存数据（毫秒级响应）
  ↓（无缓存）
执行检测 → 更新缓存 → 显示结果
```
- **优势**: 有缓存时即时显示，体验流畅
- **用户控制**: 可随时点击"重新检测"手动刷新

## 使用方法

### 1. 首次使用（无缓存）

**场景**: 应用首次运行或缓存已过期

**流程**:
1. 进入首页
2. 显示"正在检测 Python 环境..."加载状态
3. 执行环境检测（检测 Python、PyTorch、CUDA 等）
4. 显示检测结果
5. 自动缓存结果

**预期结果**:
- ✅ 看到加载状态（正常）
- ✅ 显示完整的检测结果
- ✅ 底部显示环境状态（就绪/未就绪）

### 2. 再次使用（有缓存）

**场景**: 应用之前已检测过环境

**流程**:
1. 进入首页
2. 尝试获取缓存
3. 直接显示缓存的环境信息
4. 显示"已缓存"标记

**预期结果**:
- ✅ 无加载状态，即时显示
- ✅ 显示之前的检测结果
- ✅ 显示"已缓存"标记
- ✅ 可以点击"重新检测"手动刷新

### 3. 手动刷新

**场景**: 用户想获取最新状态

**操作**:
1. 点击"重新检测"按钮
2. 按钮变为"检测中..."状态（带加载图标）
3. 执行强制刷新（绕过缓存）
4. 更新UI显示最新结果

**预期结果**:
- ✅ 显示加载状态（表示正在检测）
- ✅ 检测完成后更新显示
- ✅ 清除"已缓存"标记（表示是新数据）

### 4. 安装依赖后

**场景**: 用户通过"立即安装"安装了依赖

**流程**:
1. 点击"立即安装"
2. 在弹窗中选择安装选项（镜像、CPU/GPU）
3. 点击"开始安装"
4. 安装完成后自动触发检测
5. 验证安装结果并更新显示

**预期结果**:
- ✅ 安装完成后自动显示最新状态
- ✅ 安装的依赖显示为"已安装"
- ✅ 环境就绪时显示绿色提示

## 界面说明

### 环境检测组件位置
- **位置**: 首页的 Python 环境状态卡片
- **包含**: Python、PyTorch、torchaudio、CUDA、Ultralytics、YOLO CLI 检测

### 按钮说明
- **立即安装**: 环境未就绪时显示，点击打开安装弹窗
- **重新检测**: 始终显示，点击执行强制刷新
- **关闭**: 关闭环境检测卡片

### 状态标记
- **已缓存**: 表示显示的是之前的缓存数据，非最新检测结果
- **检测中...**: 表示正在执行检测，按钮禁用状态

### 状态图标
- ✅ **绿色勾选**: 表示该项已安装/可用
- ❌ **红色叉号**: 表示该项未安装/不可用
- ⚠️ **黄色警告**: 表示该项可选/警告（如 torchaudio、CUDA）

## 技术实现

### 缓存机制
- **存储位置**: 后端 Rust 静态变量（进程级别）
- **数据结构**: `Option<PythonEnvInfo>`
- **生命周期**: 应用运行期间持久
- **失效时机**: 
  - 手动清除缓存
  - 应用重启

### API 接口
```typescript
// 获取缓存（不执行检测）
await getCachedPythonEnv()

// 检测环境（支持缓存）
await checkPythonEnv(forceRefresh: boolean)

// 清除缓存
await clearPythonEnvCache()
```

### 缓存策略
1. **优先读取**: 进入页面先尝试获取缓存
2. **有缓存直接用**: 避免不必要的系统调用
3. **无缓存才检测**: 首次或缓存不存在时执行检测
4. **安装后刷新**: 依赖安装完成后自动更新缓存

## 常见问题

### Q1: 为什么有时候看不到"已缓存"标记？
**A**: "已缓存"标记只在有缓存且未执行新检测时显示。如果刚完成检测，会显示最新结果而不是缓存。

### Q2: 缓存会过期吗？
**A**: 当前实现没有 TTL 过期机制。缓存会一直保留到应用重启或手动清除。后续可以考虑添加过期时间。

### Q3: 安装依赖后需要手动刷新吗？
**A**: 不需要。安装完成后组件会自动触发检测并更新显示。

### Q4: 如何强制清除缓存？
**A**: 目前没有暴露清除缓存的 UI。如果需要，可以通过刷新页面或重启应用来清除。

### Q5: 为什么卸载依赖后显示的还是之前的状态？
**A**: 因为显示的是缓存数据。点击"重新检测"按钮可以获取最新状态。

## 后续优化建议

### 1. 持久化缓存
- 将缓存写入本地文件（如 JSON）
- 应用重启后恢复缓存
- 减少冷启动时的检测

### 2. 缓存过期机制
- 添加 TTL（如 5 分钟）
- 定期自动刷新
- 平衡实时性和性能

### 3. 部分检测缓存
- 仅缓存耗时长的检测项
- 快速检测项实时检测
- 进一步优化体验

### 4. 后台静默刷新
- 缓存命中时在后台静默刷新
- 不阻塞 UI 显示
- 检测完成自动更新
- 注意：当前实现已移除此逻辑，因为用户体验不佳
