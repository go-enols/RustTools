# 桌面推理性能分析指南

## 概述

本指南将帮助您分析桌面推理的性能瓶颈，找出为什么推理速度很慢（几分钟才推理出一帧）。

## 快速开始

### 步骤 1：查看性能日志

1. 启动 Tauri 开发服务器：
   ```powershell
   npm run tauri dev
   ```

2. 进入桌面推理页面
3. 选择一个模型文件
4. 点击"开始捕获"

### 步骤 2：观察性能日志

当开始捕获后，终端会输出详细的性能日志，格式如下：

```
[PERF-Frame] ===== 第 1 帧 =====
[PERF-Capture] 屏幕捕获: 25.3ms (分辨率: 2560x1600)
[PERF-Resize] Resize到640x640: 12.1ms
[PERF-Inference] 推理总耗时: 2000.0ms
  [PERF-Inference] 预处理: 15.2ms
  [PERF-Inference] 模型推理: 1950.0ms
  [PERF-Inference] 后处理 (找类+NMS过滤): 30.5ms (初始检测数: 100)
  [PERF-Inference] NMS: 5.2ms (最终检测数: 3)
[PERF-Draw] 画 3 个框: 8.5ms
[PERF-Encode]
  Resize: 2.1ms | JPEG: 85.3ms (45.2KB) | Base64: 120.5ms (60.1KB) | 总计: 207.9ms
[PERF-Emit] 发送帧: 5.2ms
[PERF-Frame] 第 1 帧总计: 2259.1ms
```

### 步骤 3：分析性能指标

#### 1. FPS (实际帧率)

观察终端输出的 FPS 信息：
```
[Desktop] 📊 实际 FPS: 0.44 | 帧处理时间: 2272.73ms
```

- **FPS < 5**: 严重性能问题 ❌
- **FPS < 10**: 中等性能问题 ⚠️
- **FPS >= 10**: 基本可接受 ✅

#### 2. 各阶段耗时分析

| 阶段 | 正常耗时 | 问题阈值 | 可能原因 |
|------|---------|----------|----------|
| **屏幕捕获** | 10-30ms | >50ms | 屏幕分辨率过高 |
| **Resize** | 5-15ms | >30ms | 算法不够快 |
| **模型推理** | 20-100ms | >200ms | ONNX模型问题、Tract性能 |
| **后处理** | 10-50ms | >100ms | 检测数量过多 |
| **NMS** | 5-20ms | >50ms | 候选框过多 |
| **JPEG编码** | 10-50ms | >100ms | 质量过高 |
| **Base64编码** | 20-100ms | >200ms | 图像太大 |
| **帧发送** | 1-10ms | >50ms | 网络问题 |

#### 3. Base64 编码问题

Base64 编码会增加 **33%** 的数据大小，如果 Base64 耗时过长，说明图像太大。

## 常见性能问题

### 问题 1：模型推理时间过长

**症状：**
```
[PERF-Inference] 模型推理: 1950.0ms
```

**可能原因：**
1. ONNX 模型太大（>50MB）
2. 使用的是 CPU 而不是 GPU
3. Tract 推理引擎性能问题
4. 模型格式不兼容

**解决方案：**
- 使用更小的模型（yolo11n 而非 yolo11x）
- 启用 GPU 加速
- 尝试不同的 ONNX 优化选项
- 检查模型输出格式是否正确

### 问题 2：Base64 编码过慢

**症状：**
```
[PERF-Encode] Base64: 120.5ms (60.1KB)
```

**可能原因：**
1. JPEG 图像太大
2. Base64 编码本身效率问题

**解决方案：**
- 降低 JPEG 质量（从 70 降到 40）
- 进一步缩小图像分辨率
- 使用二进制传输（WebSocket）而非 Base64

### 问题 3：屏幕捕获过慢

**症状：**
```
[PERF-Capture] 屏幕捕获: 150.3ms (分辨率: 3840x2160)
```

**可能原因：**
1. 屏幕分辨率过高（4K）
2. xcap 库性能问题

**解决方案：**
- 使用较低的捕获分辨率
- 选择较小的显示器
- 在捕获时直接 resize 到小图

### 问题 4：检测数量过多

**症状：**
```
[PERF-Inference] 后处理: 30.5ms (初始检测数: 5000)
```

**可能原因：**
1. 置信度阈值过低（0.25）
2. 模型输出太多候选框
3. 场景中有太多物体

**解决方案：**
- 提高置信度阈值（0.5-0.7）
- 添加更多的 NMS 过滤

## 性能优化优先级

### P0（立即优化）

1. **降低 JPEG 质量**（40-50）
2. **降低帧率**（10-15 FPS）
3. **提高置信度阈值**（0.5）

### P1（快速优化）

1. **添加帧跳过机制**
2. **使用二进制传输**
3. **优化前端渲染**

### P2（深度优化）

1. **模型优化**
2. **GPU 加速**
3. **并行处理**

## 性能分析文件

性能分析版本的代码保存在：
```
src-tauri/src/modules/yolo/services/desktop_capture_perf.rs
```

这个版本包含了详细的性能计时器，用于精确定位瓶颈。

## 如何使用性能分析版本

如果需要使用性能分析版本进行更深入的测试：

1. 在 `src-tauri/src/modules/yolo/services/mod.rs` 中添加：
   ```rust
   pub mod desktop_capture_perf;
   pub use desktop_capture_perf::DesktopCaptureService as DesktopCapturePerfService;
   ```

2. 在 commands 中创建对应的命令使用性能分析版本

3. 观察更详细的性能日志

4. **系统配置**（CPU、内存、GPU）
