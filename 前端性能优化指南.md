# 前端性能优化指南

## 概述

本文档提供了前端性能优化的具体方案，用于配合后端优化，提升实时检测的流畅度。

## 优化方案

### 1. 使用 Canvas 渲染（推荐）📊

相比 `<img>` 标签，Canvas 渲染更高效：

```typescript
// src/App.tsx 或相关组件
import { useEffect, useRef, useState } from 'react';

const DesktopCapture: React.FC = () => {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const [lastRenderTime, setLastRenderTime] = useState(0);
  
  useEffect(() => {
    // 目标帧率：15 FPS（与后端匹配）
    const targetFPS = 15;
    const frameInterval = 1000 / targetFPS;
    
    const handleFrame = (frame: DesktopCaptureFrame) => {
      const now = performance.now();
      
      // 帧跳过：如果距离上次渲染时间 < 帧间隔，跳过
      if (now - lastRenderTime < frameInterval) {
        return;
      }
      
      setLastRenderTime(now);
      
      // 使用 Canvas 渲染
      const canvas = canvasRef.current;
      if (!canvas) return;
      
      const ctx = canvas.getContext('2d');
      if (!ctx) return;
      
      // 创建 Image 对象
      const img = new Image();
      img.onload = () => {
        // 设置 Canvas 尺寸
        canvas.width = frame.width;
        canvas.height = frame.height;
        
        // 绘制图像（比 img.src 更新更高效）
        ctx.drawImage(img, 0, 0);
        
        // 释放内存
        URL.revokeObjectURL(img.src);
      };
      
      // 设置图像源
      img.src = `data:image/jpeg;base64,${frame.image}`;
    };
    
    // 监听事件
    const unlisten = listen<DesktopCaptureFrame>('desktop-capture-frame', (event) => {
      handleFrame(event.payload);
    });
    
    return () => {
      unlisten.then(fn => fn());
    };
  }, [lastRenderTime]);
  
  return (
    <div>
      <canvas ref={canvasRef} style={{ width: '100%', height: 'auto' }} />
    </div>
  );
};
```

### 2. 图像缓存和复用（重要）💾

避免每次都创建新的 Image 对象：

```typescript
const DesktopCaptureOptimized: React.FC = () => {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const imgCacheRef = useRef<HTMLImageElement | null>(null);
  const lastRenderTime = useRef(0);
  
  useEffect(() => {
    // 初始化时创建 Image 对象（复用）
    imgCacheRef.current = new Image();
    
    const handleFrame = (frame: DesktopCaptureFrame) => {
      const now = performance.now();
      const frameInterval = 1000 / 15; // 15 FPS
      
      // 帧跳过
      if (now - lastRenderTime.current < frameInterval) {
        return;
      }
      lastRenderTime.current = now;
      
      const canvas = canvasRef.current;
      const ctx = canvas?.getContext('2d');
      const img = imgCacheRef.current;
      
      if (!canvas || !ctx || !img) return;
      
      // 直接更新现有 Image 对象的 src
      img.src = `data:image/jpeg;base64,${frame.image}`;
      
      // 使用 onload 回调确保图像加载完成后再绘制
      const drawImage = () => {
        canvas.width = frame.width;
        canvas.height = frame.height;
        ctx.drawImage(img, 0, 0);
      };
      
      // 如果图像已经加载，直接绘制
      if (img.complete && img.naturalWidth > 0) {
        drawImage();
      } else {
        img.onload = drawImage;
      }
    };
    
    const unlisten = listen<DesktopCaptureFrame>('desktop-capture-frame', (event) => {
      handleFrame(event.payload);
    });
    
    return () => {
      unlisten.then(fn => fn());
    };
  }, []);
  
  return <canvas ref={canvasRef} />;
};
```

### 3. Web Worker 渲染（高级）🚀

将图像渲染放到 Web Worker 中，避免阻塞主线程：

```typescript
// src/workers/imageRenderer.worker.ts
self.onmessage = (event) => {
  const { imageData, width, height } = event.data;
  
  // 创建离屏 Canvas
  const canvas = new OffscreenCanvas(width, height);
  const ctx = canvas.getContext('2d');
  
  // 创建 ImageBitmap（比 Image 更高效）
  const blob = new Blob([imageData], { type: 'image/jpeg' });
  createImageBitmap(blob).then((bitmap) => {
    ctx.drawImage(bitmap, 0, 0);
    
    // 返回 CanvasRenderingContext2D
    const context = canvas.getContext('2d');
    self.postMessage({ success: true });
    
    bitmap.close();
  });
};

// src/App.tsx
const worker = new Worker(
  new URL('./workers/imageRenderer.worker.ts', import.meta.url),
  { type: 'module' }
);

worker.postMessage({
  imageData: frame.image,
  width: frame.width,
  height: frame.height
});
```

### 4. React 优化技巧（重要）⚛️

使用 `useMemo` 和 `useCallback` 减少不必要的渲染：

```typescript
const DesktopCaptureMemo: React.FC = () => {
  const [frame, setFrame] = useState<DesktopCaptureFrame | null>(null);
  const canvasRef = useRef<HTMLCanvasElement>(null);
  
  // 渲染逻辑用 useCallback 包裹
  const renderFrame = useCallback((f: DesktopCaptureFrame) => {
    const canvas = canvasRef.current;
    if (!canvas) return;
    
    const ctx = canvas.getContext('2d');
    if (!ctx) return;
    
    const img = new Image();
    img.onload = () => {
      canvas.width = f.width;
      canvas.height = f.height;
      ctx.drawImage(img, 0, 0);
    };
    img.src = `data:image/jpeg;base64,${f.image}`;
  }, []);
  
  // 监听事件（使用 useMemo 避免每次渲染都创建新函数）
  useEffect(() => {
    const handleFrame = (event: DesktopCaptureFrameEvent) => {
      renderFrame(event.payload);
    };
    
    listen<DesktopCaptureFrame>('desktop-capture-frame', (event) => {
      handleFrame(event.payload);
    }).then((unlisten) => {
      return () => unlisten();
    });
  }, [renderFrame]);
  
  return <canvas ref={canvasRef} />;
};
```

### 6. 请求动画帧节流（推荐）⏱️

使用 `requestAnimationFrame` 而不是 `setInterval`：

```typescript
const DesktopCaptureRAF: React.FC = () => {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const frameRef = useRef<DesktopCaptureFrame | null>(null);
  const rafIdRef = useRef<number>(0);
  
  useEffect(() => {
    // 监听事件，将帧数据存储在 ref 中
    const unlisten = listen<DesktopCaptureFrame>('desktop-capture-frame', (event) => {
      frameRef.current = event.payload;
    });
    
    // 使用 requestAnimationFrame 进行渲染循环
    const renderLoop = () => {
      const frame = frameRef.current;
      
      if (frame) {
        const canvas = canvasRef.current;
        if (canvas) {
          const ctx = canvas.getContext('2d');
          if (ctx) {
            // 渲染逻辑...
            renderFrameToCanvas(ctx, frame);
          }
        }
      }
      
      // 继续下一帧
      rafIdRef.current = requestAnimationFrame(renderLoop);
    };
    
    rafIdRef.current = requestAnimationFrame(renderLoop);
    
    return () => {
      unlisten.then(fn => fn());
      cancelAnimationFrame(rafIdRef.current);
    };
  }, []);
  
  return <canvas ref={canvasRef} />;
};
```

## 快速开始（最少代码改动）

如果不想大面积修改，可以使用这个最小优化方案：

```typescript
// 在现有的组件中添加：
const targetFPS = 15;
const frameInterval = 1000 / targetFPS;
const lastRenderTime = useRef(0);

listen<DesktopCaptureFrame>('desktop-capture-frame', (event) => {
  const now = performance.now();
  
  // 帧跳过（只渲染每 15 帧中的 1 帧）
  if (now - lastRenderTime.current < frameInterval) {
    return;
  }
  lastRenderTime.current = now;
  
  // 渲染逻辑...
  const img = new Image();
  img.src = `data:image/jpeg;base64,${event.payload.image}`;
  img.onload = () => {
    ctx.drawImage(img, 0, 0);
  };
});
```

## 常见问题

### Q: 为什么还是卡顿？
A: 检查：
1. 后端日志中的 FPS 是否为 15
2. 网络传输的数据大小（应该在 50KB 左右）
3. 是否有 JavaScript 错误
4. Canvas 尺寸是否过大

### Q: 如何调试性能？
A: 使用 Chrome DevTools：
1. Performance 面板：查看帧率
2. Network 面板：查看数据传输
3. React DevTools：查看组件渲染次数

### Q: 可以进一步优化吗？
A: 可以考虑：
1. WebSocket 二进制传输（替代 Base64）
2. WebP 编码（比 JPEG 更高效）
3. 多线程渲染（Web Worker）
4. 动态分辨率（根据网络状况调整）
