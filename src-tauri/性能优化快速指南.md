### 步骤 2：启用 scrap 零拷贝捕获

**修改 Cargo.toml：**
```toml

# xcap = "0.1"  # 可以保留，作为 fallback
```

**重新编译：**
```bash
cargo build --release
npm run tauri dev
```

**预期提升：** 捕获速度 30-50% 提升

### 步骤 4：启用 ONNX Runtime GPU 加速

**添加依赖到 Cargo.toml：**
```toml
[dependencies]
# ONNX Runtime - GPU 加速
ort = "2.0.0-rc.1"
```

**下载 ONNX Runtime（Windows）：**
```powershell
# PowerShell
$url = "https://github.com/microsoft/onnxruntime/releases/download/v1.17.0/onnxruntime-win-x64-1.17.0.zip"
Invoke-WebRequest -Uri $url -OutFile onnxruntime.zip
Expand-Archive onnxruntime.zip -DestinationPath onnxruntime
```

**设置环境变量：**
```powershell
$env:ORT_DYLIB_PATH = "D:\path\to\onnxruntime-win-x64-1.17.0\lib"
```

**重新编译并测试：**
```bash
cargo build --release
npm run tauri dev
```

**预期提升：** 推理速度 10-50x 提升（取决于 GPU）

**验证 GPU 使用：**
日志中应该显示：
```
[OnnxEngine] Device: CUDA
# 或
[OnnxEngine] Device: DirectML
```

## 故障排除

### 问题 1：scrap 编译失败

**错误：**
```
error: cannot find crate `scrap`
```

**解决：**
```toml
# 确保正确添加
scrap = { version = "0.2", features = ["xcap"] }
```

### 问题 2：ONNX Runtime 找不到库

**错误：**
```
error: failed to load onnxruntime library
```

**解决：**
1. 确认下载了正确版本的 ONNX Runtime
2. 设置环境变量 `ORT_DYLIB_PATH`
3. 或者将 ONNX Runtime DLL 复制到可执行文件目录

### 问题 3：GPU 不可用

**检查：**
```bash
# NVIDIA GPU
nvidia-smi

# ONNX Runtime
python -c "import onnxruntime as ort; print(ort.get_available_providers())"
```

**解决方案：**
- 如果没有 NVIDIA GPU，使用 DirectML（Windows 10+ 自带）
- 或者使用 CPU（性能会差一些）
